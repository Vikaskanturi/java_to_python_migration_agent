[69] 7. Agent 1 — Migration Agent
[70] File: agents/migration_agent.py
[73] 7.1 Processing Steps (per Java file)
[74] Load the file from the cloned repo path
[75] If file > 300 lines: chunk it at class/method boundaries (see core/chunker.py)
[76] For each chunk: build a prompt with class context + conversion rules and send to LLM
[77] Reassemble chunks into a single Python module
[78] Post-process: run black (formatter) + flake8 (linter) on the output
[79] Write to output/python_project/ mirroring the Java package path
[80] Log the mapping to migration_map.json

[94] File: agents/validation_agent.py
[97] 8.1 Processing Steps
[98] Load migration_state.json to get the Java→Python file pairs
[99] For each pair: send both files to LLM with the validation prompt

[102] Aggregate all mismatches into one validation_report.json

[175] 12.4 core/embeddings.py (Optional — large repos only)
[176] Enable this when the repo has more than 50 Java files. It uses a local vector store so agents can retrieve semantically relevant files as context for each migration chunk.
[177] Vector store: ChromaDB (open-source, no API key, runs locally)
[178] Embedding model: sentence-transformers/all-MiniLM-L6-v2 (runs fully offline)
[179] At startup: index all Java files into ChromaDB
[180] Per migration chunk: retrieve top-3 related files as extra context
[181] Controlled by a flag in config.yaml: embeddings: enabled: true

[199] 15. Build Notes for Anti-Gravity
[202] 15.1 Non-Negotiable Rules
[203] RULE 1 — All LLM calls go through core/llm_client.py. No agent calls any LLM SDK directly.
[204] RULE 2 — Agents communicate only through state JSON files in output/state/. No direct function calls between agents.
[205] RULE 3 — Every LLM JSON response is wrapped in try/except with a retry loop (max 3 retries, exponential backoff).
[206] RULE 4 — The GitHub ingestion module must always do a --depth=1 shallow clone to keep it fast.
[207] RULE 5 — Run black + flake8 on every generated Python file before writing it to output/.
[208] RULE 6 — Never hardcode Java→Python transformation rules in agent code. All rules live in prompt templates (prompts/*.txt).
